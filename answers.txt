Q1: Why might be consider using git?

Answer: Using git allows people to keep track of the progress of their programming. It acts as a version control for users to keep track of what works as well as add features. This can be easily done with branches to maintain a clean git tree. Git also makes it easire to work with others as their is a central code base which can be pulled and pushed code onto simplifing the development of code.

Q3: What is the difference between merging and rebasing?

Answer: Merging will create a new merge commit which will combine the changes from the branches. People use merging to keep the entire branch history of the repository. Rebasing will instead rewrite history keeping it in linear format. This makes it easire to look back at previous changes. But some people do not like rewriting history of commits.
